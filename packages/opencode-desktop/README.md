# opencode-desktop crate2nix packaging notes

This package builds `opencode-desktop` with `crate2nix`.

## Files in this directory

- `Cargo.nix`
  - generated by `crate2nix`
  - checked in on purpose
- `crate-hashes.json`
  - generated alongside `Cargo.nix`
  - records hashes for fetched git crates
- `default.nix`
  - hand-maintained package logic
  - reuses the upstream Tauri packaging inputs
  - pulls shared Tauri crate2nix compatibility shims from `outputs.lib`
- `normalize_cargo_nix.py`
  - rewrites generated source paths to use `rootSrc`

## Why the root crate override still stages repo-root assets

The Rust crate itself lives in `packages/desktop/src-tauri`, but Tauri's build
flow still expects the wider repo layout during `beforeBuildCommand` and bundle
assembly:

- `packages/desktop/package.json`
- `packages/desktop/dist`
- `packages/desktop/src-tauri/sidecars`
- vendored JS dependencies copied from the separate `opencode` package

So the crate2nix root crate:

- points `src` at the patched repo source tree
- sets `sourceRoot` to `packages/desktop/src-tauri`
- disables crate2nix's cargo vendor unpack hook
- stages the JS workspace and sidecar binary back into the repo layout during
  `preConfigure`

That keeps the Rust build rooted at the Tauri crate while preserving the wider
workspace shape the upstream desktop build expects.

## Shared Tauri helper

This package uses `outputs.lib.mkCrate2nixTauriOverrides` for the common Tauri
crate2nix compatibility layer:

- rewrites Tauri build-script env keys into shell-safe names
- copies ephemeral permission metadata into stable store paths
- patches `tauri-utils` so those rewritten keys round-trip correctly

If another Tauri crate2nix package lands in this repo, start with that helper
before adding package-local overrides.

## Version checks

- `Cargo.nix` is checked against the current `packages/desktop/src-tauri/Cargo.toml`
  version
- `default.nix` also checks that the packaged desktop app version still matches
  `packages/desktop/package.json` (allowing the existing `-<rev>` suffix)

## Regenerating crate2nix artifacts

Prefer the shared CI helper from the repo root:

```bash
nix run .#nixcfg -- ci pipeline crate2nix --write --package opencode-desktop
```

That command regenerates `Cargo.nix`, refreshes `crate-hashes.json`, runs the
package normalizer, and rewrites the generated command comment to stable
repo-relative paths.

If you need to debug the raw generation steps manually:

```bash
PATCHED_SRC=$(nix build --impure --no-link --print-out-paths \
  .#packages.aarch64-darwin.opencode-desktop.passthru.patchedSrc)

crate2nix generate \
  -f "$PATCHED_SRC/packages/desktop/src-tauri/Cargo.toml" \
  -o packages/opencode-desktop/Cargo.nix \
  -h packages/opencode-desktop/crate-hashes.json \
  --default-features

python packages/opencode-desktop/normalize_cargo_nix.py \
  packages/opencode-desktop/Cargo.nix
```

Then rebuild and adjust `crateOverrides` only if new incompatibilities show up.
